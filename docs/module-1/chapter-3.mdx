## Introduction

Topics allow ROS 2 nodes to exchange information continuously. A publisher sends data, while a subscriber receives it. Topics are ideal for sensor data streams and real-time robot behavior.

## Core Content: Topics

### Publishers and Subscribers

- A **publisher** sends messages on a topic.
- A **subscriber** listens for those messages.

### Example: Publisher

<Tabs>
<TabItem value="python" label="Python">

```python showLineNumbers title="publisher.py"
from std_msgs.msg import String
from rclpy.node import Node
import rclpy

class PublisherNode(Node):
    def __init__(self):
        super().__init__('publisher_node')
        self.publisher = self.create_publisher(String, "status", 10)
        self.timer = self.create_timer(1.0, self.publish_msg)

    def publish_msg(self):
        msg = String()
        msg.data = "Robot is running"
        self.publisher.publish(msg)
        self.get_logger().info("Message sent")

def main(args=None):
    rclpy.init(args=args)
    node = PublisherNode()
    rclpy.spin(node)
    rclpy.shutdown()

if __name__ == "__main__":
    main()
```
