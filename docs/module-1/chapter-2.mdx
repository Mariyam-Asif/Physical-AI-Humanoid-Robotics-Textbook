## Introduction

In ROS 2, nodes are the basic building blocks of a robot’s software system. Each node performs a small, focused task—such as reading a sensor or controlling a motor. Nodes communicate with one another using topics, services, and actions.

This chapter will show you how ROS 2 communication works and how different parts of a robot coordinate to create intelligent behavior.

## Core Content: ROS 2 Nodes

### What is a Node?

A **node** is an executable process responsible for one logical function.  
Examples:

- A camera node publishing images  
- A perception node detecting obstacles  
- A control node sending motor commands  

### Communication Methods

| Method | Description | Use Case |
|-------|-------------|----------|
| **Topics** | Publish/subscribe (asynchronous) | Sensor streams |
| **Services** | Request-response | Ask for a one-time result |
| **Actions** | Long tasks with feedback | Navigation, manipulation |

### Example: Creating a Simple Node

<Tabs>
<TabItem value="python" label="Python">

```python showLineNumbers title="node_example.py"
import rclpy
from rclpy.node import Node

class BasicNode(Node):
    def __init__(self):
        super().__init__('basic_node')
        self.get_logger().info("Node has started!")

def main(args=None):
    rclpy.init(args=args)
    node = BasicNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == "__main__":
    main()
```
